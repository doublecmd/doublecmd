1. Общие вопросы

В: Что такое Double Commander?

O: Double Commander — это двухпанельный файловый менеджер. Подробности можно прочитать здесь.


В: Что нового в этой версии?

O: Информацию об истории версий вы можете найти в интернете на этой странице. Но сейчас проект развивается настолько быстро, что последние ночные сборки, возможно, содержат множество новых возможностей и улучшений.


В: Это версия Total Commander® для Linux, да?

O: Нет, это независимая программа от других разработчиков. Сейчас существуют версии для ОС Windows® и Linux с графическими интерфейсами GTK2 и QT. В будущем вполне возможно появление версий программы для других операционных систем, например Mac OS.


В: В чем отличия этой программы от моего любимого Total Commander®?

O: Цель проекта — создать программу, приближенную к TC по интерфейсу, максимально совместимую с ним по плагинам и конфигурационным файлам. При этом Double Commander обладает рядом существенных преимуществ: он абсолютно бесплатен для использования, кросcплатформенный (работает под разными операционными системами), и имеет открытый исходный код.


В: У меня 64-битная ОС. Могу я скачать Double Commander, оптимизированный под мою систему?

O: Да, сборки для x64 существуют под обе поддерживаемые платформы.


В: Как можно связаться с разработчиками Double Commander?

O: Напишите в официальный форум проекта, и ваше мнение или пожелание не останется без внимания.

2. Вопросы, связанные с Linux

В: Какую версию мне скачивать?

O: Если у вас KDE — то качайте QT-сборку, иначе (Gnome, Xfce, LXDE) сборку с GTK.


В: Есть ли у проекта репозитарий?

O: Репозитория пока нет. Но вы можете скачать с официального сайта установочные пакеты deb или rpm, а также "portable" — упакованные с помощью bz или bz2 бинарники. Либо можно скачать исходный код и собрать DC самостоятельно.


В: Плагины от Тотала не работают!?

O: Работают, но не все. Плагины от TC скомпилированы под винду. Поэтому их можно подключать только к Windows-версии Double Commander. Но любой плагин, имеющий открытый исходный код, можно собрать для Линукс (или уговорить разработчика плагина это сделать :)).

наверх

3. Настройка интерфейса

В: Как поменять настройки цвета для файловых панелей?

O: Чтобы изменить цвет, надо сделать следующее:

  • Настройка → Параметры → Цвета (не получите никаких изменений, пока не создадите свой стиль колонок)

  • Настройка → Параметры → Колонки выбираете стиль по умолчанию (Default) нажимаете "Редактировать" и там все сами поймете :)


В: Для чего кнопки R и Все в настройках цвета?

O: Все очень просто:

  • R — восстанавливает цвет на тот, который был по умолчанию.

  • Все — распространяет выбранный цвет на все колонки.


В: При изменении размеров панелей колонка "Размер" налазит на колонку "Дата". Заголовок колонки изменяется нормально, а содержимое ползет...

O: В настройках на странице "Поведение" установите галочку "Обрезать текст по ширине колонки".


В: Почему заголовок табулятора, при первом запуске в linux на английском?

O: При первом запуске DC определяет язык системы на основании локали. В дальнейшем текст заголовков сохраняется в doublecmd.ini и считывается из него. Изменить заголовки можно в настройках колонок.


В: Как повесить свои значки на определенные типы файлов?

O: Можно отредактировать файл конфигурации doublecmd.ext. Пример, для связи определенного значка с графическими типами файлов:

[jpeg|jpg|gif|png]
Name=images
Icon=/home/rust/progs/doublecmd/pixmaps/32x32/mimetypes/image.png
open=gthumb '%p'

В этом случае, все перечисленные выше расширения будут отображаться со значком image.png.


В: ERROR: [TGtkWidgetSet.GetSysColor] Bad Value: 45. Valid Range between 0 and 31. Что это?

O: Это известная ошибка, она связана с переходом на новую версию Lazarus IDE (точнее, библиотеки LCL). Чтобы избавиться от этой ошибки, нужно в диалоге настроек на странице "Цвета" → "Файловые панели" заново выбрать цвет "Текст под курсором", а также в "Настройка" → "Параметры" → "Колонки" проделать то же самое.


В: Как создать разделитель на панели кнопок?

O: Чтобы получить разделитель, необходимо в качестве подсказки указать "-".

4. Вопросы, связанные с навигацией и управлением

В: Что надо сделать, чтобы при выделении пробелом, курсор сдвигался вниз на одну строчку?

O: Установить в файле doublecmd.ini в секции [Configuration] ключ

SpaceMovesDown = 1

В: Есть ли здесь возможность создавать очередь копирования файлов?

O: В настоящее время нет. Будет реализовано в ближайшем будущем.


В: Ctrl+A не работает, где включить?

O: Дело в том, что настройки по умолчанию для хоткеев применяются в случае отсутствия shortcuts.ini (например при первом запуске после установки), так как данный файл уже существует новые настройки не применились. Включить можно так:

  • в диалоге настроек на странице "Горячие клавиши" установить данный хоткей для команды cm_MarkMarkAll

  • в файл shortcuts.ini добавить следующие строки:

    [Ctrl+A]
Command0=cm_MarkMarkAll
Object0=frmMain
Form0=frmMain


В: Скопировать папку с переименованием не получается.

O: Ситуация с переименованием директории на данный момент не обрабатывается. Возможность будет реализована позже.


В: Тут можно создавать выпадающее меню? Как это делать?

O: Можно, добавляем на панель кнопку, в качестве команды выбираем cm_ShowButtonMenu, в параметры прописываем путь к *.bar файлу (если он находится в каталоге с конфигами, то достаточно указать только имя файла).


В: Как можно настроить контекстное меню? Например добавить пункт "Открыть с помощью"?

O: Это можно сделать в ассоциации файлов вручную, обозначив какие расширения какими программами можно открывать, там же назначить какое приложение будет использоваться по умолчанию.

"Настройка" — "Файловые ассоциации" — "Типы файлов" (вводите любое имя, например media) — "Расширения" (добавить, например avi) — "Команды" — "Добавить" (действие пишите open — "Команда" пишите, например totem-xine "%p" или mplayer "%p" или smplayer "%p" и т. д.). После этих действий по правому клику можно будет выбирать чем открыть файл.

Кстати, можно легко сделать так, чтобы любую папку можно было открыть, например, в Наутилусе (Gnome) с правами рута. Чтобы это сделать, надо добавить в ассоциации новый тип файлов (например dir), а в качестве расширения указать folder, в качестве действия Open as root и вписать команду: gksu nautilus "%p". Правой кнопкой на папке, в контекстном меню выбираете Open as root и вводите пароль :)

наверх

5. Вопросы, связанные со встроенной программой просмотра (F3

В: Окна редактора и просмотрщика появляются где хотят, при включенном Compiz'е.

O: В настройках Compiz: place windows → windows with fixed positions и window rules → fixed size windows.

Код: (class=Doublecmd) & (title=/)

В: Во встроенном редакторе странный шрифт — по горизонтали символы друг на друга наезжают.

O: Необходимо использовать моноширинные шрифты как во вьювере, так и редакторе (ограничение используемого компонента).


В: На одну единственную картинку вьювер не срабатывает. Остальные картинки просматриваются, а вот эту отказывается.

O: Судя по всему, компонент используемый для просмотра изображений не поддерживает некоторые JPG файлы. Теперь вместо ошибки такие файлы открываются как текст.


В: Стало неожиданостью... смотрелка не видит .gif :(

O: Да это так, однако компонент для отображения GIF есть, осталось только его прикрутить к просмоторщику. Будет реализовано в ближайшее время.


В: Есть такие текстовые файлы, в которых при быстром просмотре строчки уходят за край окошка! А внизу нет ползунка чтобы прочесть до конца!

O: Хм... действительно горизонтальный ползунок не предусмотрен, в данном случае можно воспользоваться режимом "В виде текста с разрывами строк".

наверх
